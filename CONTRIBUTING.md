# WIKI 기여 가이드

이 문서는 WIKI 레포에 학습 내용을 작성할 때 따라야 할 규칙을 정리합니다.
충돌 없이 여러 사람이 동시에 작성할 수 있도록, 아래 구조와 규칙을 지켜주세요.

---

## 1. 폴더 구조

```
docs/
  cs-기초/              # L0: 카테고리
    README.md           # 카테고리 목차 (키워드 목록 + 상태)
    32bit-64bit.md      # L1: 키워드 파일
    memory-structure.md
    ...
  algorithm/
  malloc-lab/
  네트워크/
  os/
  pintos/
    project0/           # L1: 하위 폴더
      README.md          # L1 목차
      virtual-machine.md # L2: 키워드 파일
      ...
    project1-threads/
    ...
  ai/
  db/
  web-security/
  팀운영/
    컨벤션/
    외부리소스/
  meeting-minutes/
    week9/
      2026-04-25.md
```

최대 깊이는 L0 ~ L9 (10레벨)까지 허용합니다.
하위 레벨이 필요하면 폴더를 만들고 README.md로 목차를 작성하세요.

---

## 2. 파일 네이밍 규칙

- 영문 소문자, 하이픈(-) 사용: `cpu-vs-gpu.md`, `hash-table.md`
- 한글 카테고리명은 허용: `cs-기초/`, `네트워크/`
- 공백 대신 하이픈: `deep-shallow-copy.md` (O), `deep shallow copy.md` (X)
- 각 폴더에 반드시 `README.md` 포함 (해당 폴더의 목차 역할)

---

## 3. 키워드 파일 템플릿

새 키워드 파일을 만들 때는 아래 템플릿을 따르세요:

```markdown
# 키워드 제목

> 한 줄 요약 설명

## 개요

_작성 필요_

## 상세 설명

_작성 필요_

## 참고 자료

_링크 추가_
```

필요에 따라 아래 섹션을 추가할 수 있습니다:
- `## 코드 예시`
- `## 비교 표`
- `## 생각해볼 질문`
- `## 관련 키워드` (다른 파일 링크)

---

## 4. README.md (목차) 템플릿

각 폴더의 README.md는 아래 형식을 따릅니다:

```markdown
# 카테고리 이름

카테고리 설명 한 줄.

| 키워드 | 파일 | 상태 |
|--------|------|------|
| 키워드 이름 | [파일명.md](파일명.md) | 작성 필요 / 작성 중 / 완료 |
```

---

## 5. 작성 상태 관리

각 README.md의 상태 컬럼을 업데이트하세요:
- `작성 필요` — 아직 내용이 없음
- `작성 중` — 누군가 작성 중 (본인 이름 표기 권장)
- `완료` — 내용 작성 완료

---

## 6. 충돌 방지 규칙

1. **하나의 키워드 파일은 한 사람만 작성합니다.** 작성을 시작하면 해당 README.md의 상태를 `작성 중 (@이름)`으로 변경하고 커밋하세요.
2. **README.md(목차)는 키워드 추가/상태 변경 시에만 수정합니다.** 내용 작성은 개별 키워드 파일에서 합니다.
3. **새 키워드를 추가할 때:** 키워드 파일 생성 + README.md에 행 추가를 하나의 커밋으로 묶으세요.
4. **하위 레벨 폴더를 추가할 때:** 폴더 생성 + README.md + 상위 README.md 링크 추가를 하나의 커밋으로 묶으세요.

---

## 7. 커밋 메시지 규칙

```
docs(<카테고리>): <작업 내용>
```

예시:
```
docs(cs-기초): 32bit-64bit 키워드 작성
docs(pintos/project1): scheduler 키워드 작성 시작
docs(meeting-minutes): week9 2026-04-25 회의록 추가
```

---

## 8. 브랜치 전략

- `main` 브랜치에 직접 커밋합니다 (문서 레포이므로).
- 대규모 구조 변경 시에는 브랜치를 만들어 PR을 올려주세요.

---

## 9. 하위 레벨 추가 방법

키워드가 복잡해서 여러 파일로 나눠야 할 때:

1. 기존 키워드 파일을 폴더로 변환합니다
2. 폴더 안에 README.md(목차) + 하위 키워드 파일을 만듭니다

예시: `scheduler.md` → `scheduler/README.md` + `scheduler/round-robin.md` + `scheduler/priority.md`

---

## 10. 회의록 규칙

- 위치: `docs/meeting-minutes/week{N}/YYYY-MM-DD.md`
- 형식: 기존 TEMPLATE.md를 따릅니다
- 파일명: 날짜 기준 (`2026-04-25.md`)
