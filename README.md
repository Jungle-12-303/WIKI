# Jungle 12-303 WIKI

크래프톤 정글 12기 303호의 공동 학습 기록 저장소입니다.

팀원 모두가 동일한 키워드 체계 아래 학습한 내용을 정리하고, 서로의 글을 참고하며 함께 성장하는 것을 목표로 합니다.

## 이 저장소의 역할

이 저장소는 개인 노트가 아닙니다. 반 전체가 공유하는 **공동 지식 베이스**입니다.

- 정해진 키워드 체계를 따라 같은 위치에 같은 형식으로 학습 내용을 쌓습니다.
- 누가 작성하든 다른 사람이 읽고 이해할 수 있어야 합니다.
- 한 키워드를 한 사람이 맡아 작성하고, 상태를 명시해서 충돌을 막습니다.

## 빠른 시작 (온보딩)

### 1단계: 레포 클론

```bash
git clone https://github.com/Jungle-12-303/WIKI.git
cd WIKI
```

### 2단계: 구조 파악

```
docs/
  cs-기초/           CS 기본 개념 (메모리, 포인터, 보수 표현 등)
  algorithm/         알고리즘 및 자료구조
  malloc-lab/        동적 메모리 할당기 구현
  네트워크/           TCP/IP, HTTP, REST API, OSI 7계층
  os/                운영체제 핵심 개념
  pintos/            Pintos 프로젝트 (Project 0~4)
    project0/        환경 설정
    project1-threads/
    project2-userprog/
    project3-vm/
    project4-filesys/
  ai/                신경망, LLM, AI 응용기술
  db/                데이터베이스
  web-security/      웹 개발 & 보안
  팀운영/
    컨벤션/           Git, 코드, PR 규칙
    외부리소스/        개발 환경, 도구, 학습 자료
  meeting-minutes/   팀장 회의록
```

각 폴더의 `README.md`가 해당 카테고리의 목차입니다. 여기서 작성 상태를 확인하세요.

### 3단계: 작성할 키워드 선택

1. 관심 있는 카테고리 폴더의 `README.md`를 엽니다.
2. 상태가 `작성 필요`인 키워드를 고릅니다.
3. 해당 키워드 파일을 열어 내용을 작성합니다.

### 4단계: 상태 업데이트 및 커밋

1. 작성을 시작하면 해당 카테고리의 `README.md`에서 상태를 변경합니다:

```markdown
| CPU vs GPU | [cpu-vs-gpu.md](cpu-vs-gpu.md) | 작성 중 (@홍윤기) |
```

2. 커밋합니다:

```bash
git add docs/cs-기초/cpu-vs-gpu.md docs/cs-기초/README.md
git commit -m "docs: CPU vs GPU 키워드 작성 시작"
git push origin main
```

3. 작성이 끝나면 상태를 `완료`로 변경하고 다시 커밋합니다.

### 5단계: 새 키워드 추가 (필요한 경우)

기존 목록에 없는 키워드를 추가하고 싶다면:

1. 적절한 카테고리 폴더에 md 파일을 생성합니다.
2. 같은 폴더의 `README.md` 표에 행을 추가합니다.
3. 두 파일을 하나의 커밋으로 묶습니다.

파일명 규칙: 영문 소문자 + 하이픈 (`cpu-vs-gpu.md`). 공백이나 대문자는 사용하지 않습니다.

## 기여 규칙 요약

자세한 규칙은 [CONTRIBUTING.md](CONTRIBUTING.md)를 참고하세요. 핵심만 정리하면:

| 규칙 | 내용 |
|------|------|
| 하나의 파일, 한 사람 | 같은 키워드 파일을 두 사람이 동시에 수정하지 않습니다 |
| 상태 먼저 변경 | 작성 시작 전에 README.md 상태를 `작성 중 (@이름)`으로 바꾸고 커밋 |
| 커밋 메시지 | `docs: <한국어 제목>` 형식 (커밋 컨벤션 참고) |
| 파일명 | 영문 소문자 + 하이픈, 공백 금지 |
| 키워드 템플릿 | `# 제목` → `> 요약` → `## 개요` → `## 상세 설명` → `## 참고 자료` |

## 커밋 컨벤션

이 레포는 [Conventional Commits](https://www.conventionalcommits.org/) 형식을 따르되, 제목과 본문은 한국어로 작성합니다.

```
<type>: <한국어 제목>
```

주로 사용하는 타입:

| 타입 | 용도 |
|------|------|
| `docs` | 키워드 작성, 문서 변경 (가장 많이 사용) |
| `feat` | 새로운 스크립트나 도구 추가 |
| `fix` | 오류 수정 |
| `chore` | 폴더 구조 변경, 설정 변경 |

예시:

```
docs: CPU vs GPU 키워드 작성
docs: week10 팀장 회의록 추가
chore: pintos/project2 폴더 구조 추가
```

제목은 구체적으로 씁니다. `수정`, `업데이트`, `작업` 같은 막연한 단어는 피합니다.
